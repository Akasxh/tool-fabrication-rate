# Who Hallucinates Tools, How Often, and What Fixes It

A per-call study of tool-name fabrication in tool-using LLM agents, with a training-free fix.

> **Who Hallucinates Tools, How Often, and What Fixes It: A Per-Call Study Across Anthropic 4.x and Qwen3**
> S Akash and Shine Gupta 
> Accepted at **SCALE @ ICML 2026**.
> Paper: [`paper/main.pdf`](paper/main.pdf)

This repository contains the experimental harness, run scripts, aggregated results, and LaTeX source backing the paper. It defines a per-call **Tool Fabrication Rate (TFR)** — the fraction of tool calls that name a tool absent from the provided registry — measures it across commercial and open-weight model families on BFCL multi-turn (and a τ-bench retail cross-check), and proposes **Registry-Visible Reprompting (RVR)**, a training-free middleware that eliminates the fabrications.

> Note: some scripts and trace fields use the earlier name **TEHR** for what the paper calls **TFR**; they refer to the same per-call metric.

## TL;DR of the findings

- **Commercial models do not fabricate.** Across Anthropic 4.x (five versions) and OpenAI gpt-4.1 / gpt-4o tiers, **0 fabrications over ~6,000 opaque-baseline probes** (Anthropic 0/2,592, two-sided 95% upper bound ≤0.14%; OpenAI 0/2,117 pooled, ≤0.18%). The gpt-5 generation (gpt-5 / mini / nano / 5.5) extends the null as a confirmatory check (**0/1,311 pooled, all 0% TFR**), kept outside the locked headline pool.
- **Open-weight families do.** Qwen3 logs non-zero TFR, peaking at **1.64% (14B)**; across open lineages the worst per-arm rate reaches **~10.4% (Qwen2.5)**. Llama-3.1-8B reproduces the gap, and a 4-bit/8-bit/bf16 precision ladder shows it is not a quantization artefact.
- **The fabrications are dangerous.** About **70% name state-changing actions** carrying concrete arguments (e.g. fabricated `place_order`, `cancel_booking`), so under a permissive executor the failure is an unintended *action*, not a failed lookup.
- **It is a grounding failure, not a content failure.** On a high-event cell, a *wrong* registry list recovers as well as the real one (Fisher exact **p = 1.0**) — recovery is format-driven.
- **RVR fixes it.** On a registry miss, RVR returns one re-prompt wrapping a structured `tool_not_found` envelope. It removes **all 20 pooled Qwen3 fabrications** in the matched per-tier subset (Fisher exact one-sided, **p = 3.5×10⁻⁵**, 20/1,417 vs 0/945), with no training.

## Installation

The harness is a [uv](https://docs.astral.sh/uv/)-managed Python package targeting **Python 3.12**. The virtual environment lives at `harness/.venv`.

```bash
# From the repo root
uv venv harness/.venv --python 3.12
uv pip install --python harness/.venv/bin/python -e ./harness
```

Dependencies (Anthropic + OpenAI SDKs, `mlx-lm`, scipy/numpy/statsmodels, etc.) are pinned in [`harness/pyproject.toml`](harness/pyproject.toml).

**API keys.** Commercial models require provider credentials. Put them in `.env.local` at the repo root (gitignored):

```
ANTHROPIC_API_KEY=...
OPENAI_API_KEY=...
```

**Local / open-weight models** run via **MLX on Apple Silicon** (e.g. `mlx-community/Qwen3-8B-4bit`); no API key is needed, and the precision ladder uses 4-bit / 8-bit / bf16 MLX builds.

## Quickstart

All commands run from the repo root with `PYTHONPATH=.` so the `harness` package resolves.

Run the distractor-injection probe on a model. This injects one synthetic distractor tool (`near_name` / `synonym` / `matched_random` / `unrelated`) into each BFCL multi-turn registry and measures TFR:

```bash
# Open-weight model, opaque baseline, with the real target tool removed
# (Czapla-style: the model expects a tool the registry no longer offers)
PYTHONPATH=. harness/.venv/bin/python scripts/run_probe.py \
    --models mlx-community/Qwen3-8B-4bit \
    --distractor-types near_name,synonym,matched_random,unrelated \
    --n 25 \
    --condition C0 \
    --remove-target
```

```bash
# Commercial model, RVR intervention on (condition C1)
PYTHONPATH=. harness/.venv/bin/python scripts/run_probe.py \
    --models claude-haiku-4-5,gpt-4.1 \
    --n 25 \
    --condition C1
```

Key flags (see `scripts/run_probe.py` for the full argparse):

| Flag | Values | Meaning |
|---|---|---|
| `--models` | comma-separated IDs | `claude-*` → Anthropic, `gpt-*`/`grok-*` → OpenAI SDK, anything else → MLX |
| `--distractor-types` | `near_name,synonym,matched_random,unrelated` | which synthetic distractor(s) to inject |
| `--condition` | `C0, C0_5, A2, C0_7, C0_8, C1` | C0 = opaque baseline, C0.5 = naive retry, C1 = RVR (full registry list); C0.7/C0.8 are the content ablation arms |
| `--remove-target` | flag | also remove the real tool the distractor mimics |
| `--benchmark` | `bfcl, tau_bench` | BFCL multi-turn (default) or τ-bench retail |
| `--n` | int | base tasks per cell |
| `--seed` | int | deterministic task sampling |
| `--run-id` / `--output` | str | run identifier and output root (default `results/`) |

> Note: the quickstart above uses `--n 25` (the `run_probe.py` default). The paper's **locked distractor probe used `--n 15` per cell**, so pass `--n 15` to match the reported probe N; `--n 25` is fine for a larger smoke run but will not reproduce the paper's exact probe counts.

Each run writes one JSONL trace per cell under `results/<run-id>/` (schema in `harness/HARNESS_SPEC.md` §3), plus a `repro_manifest.json`.

## Reproducing the paper numbers

> Important: the trace directories (`results/`, `paper/results/`) and the aggregator output directory (`PHASE0/RESULTS/`) are **gitignored**, so a fresh clone of this repo contains **no traces to aggregate** and **none of the named output files below**. You must generate the traces first (see Quickstart and the `run_*.sh` drivers) before `aggregate_all.py` can reproduce the numbers. The files under `PHASE0/RESULTS/` are regenerated locally by the aggregator — they are not shipped artifacts in the repo.

Once traces exist under `results/`, `aggregate_all.py` walks every JSONL trace (under `results/` and `paper/results/`), recomputes the per-cell TFR, the Clopper–Pearson bounds, and the headline statistics, and writes them out:

```bash
# 1. Generate traces first (Quickstart probe and/or the batch drivers):
PYTHONPATH=. harness/.venv/bin/python scripts/run_probe.py --models ... --n 15 --condition C0
#    or the larger sweeps:
bash scripts/run_queue.sh   # (and the other run_*.sh drivers)

# 2. Then aggregate:
PYTHONPATH=. harness/.venv/bin/python scripts/aggregate_all.py
```

The aggregator (re)generates these locally under `PHASE0/RESULTS/`:

- `headline_numbers.json` / `headline_numbers.md` — the per-model/per-arm rates that fill the paper's reported numbers.
- `stats_report.json` / `stats_report.md` — Fisher exact, McNemar, TOST, and confidence-interval reports (also via `scripts/run_stats.py` / `scripts/run_stats_full.py`).

The batch `run_*.sh` drivers in `scripts/` reproduce the larger sweeps (frontier tiers, family breadth, the τ grid, and the precision ladder) that populate `results/` before aggregation.
